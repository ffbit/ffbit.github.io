HTML Minification in Jekyll

Mar 17, 2021

[ html minification, jekyll ]

One way to improve a web site’s performance is to reduce the size of files transferred over HTTP by removing non-functional information (comments, indentation, etc.). This process is called minification[1] and is often applied to CSS, JS, and HTML.

Out of the box, Jekyll allows minifying CSS with its jekyll-sass-converter[2]. As for the HTML minification, there is a well-known, standard way to do so in Jekyll by means of the extensively documented compress layout[3],[4],[5].

The Problem

However, when it came to minifying HTML with code snippets containing line numbers for this site, I discovered that it produced incorrect formatting caused by the apparent incompatibility between jekyll-compress-html and non-conforming HTML produced by Jekyll[6].

The screenshots taken from[6] demonstrate the issue:

Not only the line numbers and code block were misaligned, but also the site’s footer looked to belong to the code block. Other sources related to the problem[7],[8],[9],[10].

A Solution

An ad hoc Jekyll layout based on htmlcompressor[11] can solve the issue of the HTML minification with code blocks in Jekyll. Here are the steps.

In the Gemfile, add the htmlcompressor’s dependency

1

gem 'htmlcompressor', '~> 0.4.0'

and install it

1

bundle install

Then, create a Jekyll filter plugin[12], say, _plugins/minify.rb

require 'htmlcompressor'

module Jekyll
    module MinifyFilter
        def minify(content)
            if minify_enabled?
                compressor = HtmlCompressor::Compressor.new
                compressor.compress content
            else
                content
            end
        end

        private

        def minify_enabled?
            config = @context.registers[:site].config
            true?(config['minify']) and Jekyll.env === 'production'
        end

        def true?(obj)
            obj.to_s.downcase == 'true'
        end
    end
end

Liquid::Template.register_filter(Jekyll::MinifyFilter)

which is enabled when both minify property is set to true in _config.yml

1

minify: true

add Jekyll environment is set to production

1

$ JEKYLL_ENV=production jekyll build

Finally, create a Jekyll layout _layouts/minify.html

1

{{ content | minify }}

and use it as the parent layout in _layouts/default.html

1
2
3

---
layout: minify
---

That’s it! Check this page’s source to see the result. Notice that the HTML of code snippets is not minified.
